# Contributing

Thanks for helping improve Mauritius Mall.

## Development workflow

1. Create a feature branch from `main`.
2. Keep changes focused and easy to review.
3. Prefer small, well-documented commits.
4. Run the relevant checks before opening a PR.

### Branch naming

Use descriptive branch names such as:

- `feature/product-filters`
- `fix/checkout-validation`
- `chore/update-docs`

### Pull request checklist

- Clear title and summary
- Screenshots for UI changes when relevant
- No secrets or local environment values committed
- Documentation updated when behavior changes

## Code quality expectations

- Follow existing TypeScript and React patterns
- Keep components small and reusable
- Prefer clear naming over clever abstractions
- Add or update documentation when adding new features
